# Contributing

Thanks for your interest in improving this project.

Beginner-friendly contributions are welcome, including:

- improving the README
- adding examples
- improving error messages
- testing on Windows, macOS, or Linux
- suggesting better matching rules
- reporting bugs

## How to contribute

1. Open an issue describing the problem or idea.
2. Fork the repository.
3. Create a branch.
4. Make your changes.
5. Submit a pull request.

## Good issue reports include

- your operating system
- your Python version
- a small example of the Excel mapping
- a small example of the original filename
- the error message or generated report
